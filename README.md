# Maritime Crew Readiness and Deployment Analytics Dashboard

## Project Overview

This is a company-inspired data analyst case study for a maritime manning agency. The project uses a simulated crew dataset to analyze crew readiness, deployment status, document expiration risks, contract timelines, and department-level staffing needs.

## Business Problem

Maritime manning agencies need to monitor crew deployment readiness, document validity, contract timelines, and manpower availability. Manual tracking can make it difficult to quickly identify which crew members are ready for deployment and which ones need document renewal or follow-up.

## Project Objective

The objective of this project is to analyze simulated crew records and identify:

* Total crew by department
* Crew deployment status
* Crew readiness status
* High-risk or expired documents
* Departments with the highest document-related risks
* Crew members ready for deployment

## Dataset

The dataset used in this project is simulated and does not contain real company, employee, or seafarer data.

## Tools Used

* Python
* Pandas
* SQLite
* Structured Query Language
* Excel
* Power BI planned for dashboard visualization

## Key Files

* `data/maritime_crew_readiness_dataset.csv` - raw simulated dataset
* `data/cleaned_maritime_crew_data.csv` - cleaned dataset with calculated readiness and document risk columns
* `sql/maritime_crew_analysis_queries.sql` - Structured Query Language queries used for analysis
